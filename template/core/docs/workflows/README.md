# workflows

User-journey / feature-spec documents: what a user is trying to do, the happy path,
edge cases, business rules, and data touchpoints — written so the journey layer can be
confirmed with non-technical stakeholders. These outrank UAT docs in the source-of-truth
precedence when they exist.

Create a document here when a feature's behavior is being specified; keep it current in
the same PR as behavior changes (documentation standard's keep-current rule).
